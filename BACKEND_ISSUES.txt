BACKEND SETUP — READ THIS FIRST
================================
Written for whoever is picking up backend (Task 4). Use Claude to fix these.
Issues are in priority order — top 2 block the server from starting at all.


CRITICAL (server won't run without fixing these)
-------------------------------------------------

1. PORT CONFLICT
   NEMORON_URL defaults to http://localhost:8000 in .env.example
   FastAPI also runs on 8000. They clash.
   Fix: run the model server on 8001, set NEMORON_URL=http://localhost:8001 in your .env

2. NO .env FILE
   Only .env.example exists. Server won't start.
   Fix: cp backend/.env.example backend/.env
   Then fill in DATABASE_URL, NEMORON_URL, MAPBOX_TOKEN

3. SPATIAL TABLES DON'T EXIST IN POSTGRES
   spatial.py queries tables: traffic_volumes, ttc_stops, street_trees,
   business_licences, parks, zoning_bylaws
   These tables are never created — the data pipeline is in omar/data branch (not merged).
   Fix: merge omar/data branch, then write a load_spatial.py script that reads
   the parquet files from data/ and writes them into PostGIS.
   Until this is done, all spatial queries return empty lists (fallback still works).


MEDIUM (bad UX but server runs)
--------------------------------

4. IMPACT ENDPOINT BLOCKS FOR ~20s
   GET /building/{id}/impact holds the HTTP connection open while NeMoTron runs.
   Frontend will appear frozen.
   Fix: change to POST /building/{id}/analyze → returns 202 + job_id,
   then GET /building/{id}/impact polls for result.

5. NO DELETE /building/{id} ENDPOINT
   Builder UI needs "Clear Building" to remove a placed building from DB.
   Fix: add a 2-line delete route to routers/buildings.py

6. ChatSession.building_id FK exists in the model but router never sets it
   Fix: pass building_id when creating the session in chat.py


MINOR
------

7. No startup script
   Fix: add backend/run.sh with:
     uvicorn main:app --host 0.0.0.0 --port 8000 --reload

8. NEMORON_URL typo in variable name (missing T) — works fine, just inconsistent


MODEL ENDPOINT NOTE
-------------------
If running llama.cpp server locally (like the Qwen setup):
  - llama.cpp's server already exposes an OpenAI-compatible /v1/chat/completions endpoint
  - Set NEMORON_URL to wherever that server is running
  - Change the model name in agents/impact_agent.py and agents/chat_agent.py
    to match whatever model name llama.cpp reports (usually the filename or "local-model")

If using NVIDIA Build API (cloud fallback):
  - Set NEMORON_URL=https://integrate.api.nvidia.com/v1
  - Add NGC_API_KEY to .env
  - Add Authorization header to the httpx calls in both agent files:
    headers={"Authorization": f"Bearer {os.getenv('NGC_API_KEY')}"}
